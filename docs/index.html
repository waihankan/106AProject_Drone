3. Hand Gesture Control

Hand Gesture Control leverages MediaPipe for real-time hand tracking, enabling intuitive drone control through natural gestures. The system detects pinch gestures to activate following mode.

3.1. Gesture Recognition

MediaPipe tracks 21 hand landmarks in real-time, with the system specifically monitoring the distance between thumb tip and index finger tip to detect pinch gestures. This provides a reliable and intuitive control interface.

3.2. Tracking Behavior

The hand gesture mode operates in "relative mode," meaning the drone continuously adjusts to follow hand movements without a fixed reference distance. This creates a dynamic, responsive control experience.

Technical Specifications

  • Single hand tracking (max 1 hand detected)
  • 21 landmark points per hand
  • Pinch threshold dynamically calculated
  • Real-time processing at ~30 FPS
  • Immediate stop on gesture release (safety feature)

4. Follow Me Mode

Follow Me Mode combines vision detection with intelligent control algorithms to autonomously track targets. When the target is lost, the system automatically initiates search behavior with spiral waypoint navigation.

4.1. Following Controller

The FollowController implements a sophisticated tracking algorithm with two primary objectives: centering the target in the frame and maintaining desired distance. The system uses priority-based control, first ensuring the target is centered before adjusting distance.

Control Features

Exponential Moving Average (EMA)

4.2. Search and Recovery

When the target is lost, the system automatically switches to search mode, rotating the drone in 30-degree increments to locate the target. If the target remains undetected, spiral waypoint navigation is initiated.

5. ArUco Marker Mode

The ArUco Marker Mode provides precise 3D pose estimation using calibrated camera parameters. This mode is ideal for applications requiring accurate distance measurement and stable tracking performance.

5.1. Detection and Calibration

The system uses OpenCV's ArUco detector with camera calibration matrices to estimate the marker's 3D position. Before flight, the user must perform a calibration by holding the marker within the deadzone for 2 seconds to establish a reference distance.

Key Features

  • check_circle 3D Pose Estimation: Translation vectors (tvec) provide accurate distance and position data
  • check_circle Deadzone Logic: 30% central region prevents jitter and ensures stability
  • check_circle Absolute Reference: Maintains fixed distance based on initial calibration
  • check_circle Marker Size: 13cm markers optimized for detection range

5.2. Control Algorithm

The controller uses proportional gains to center the marker and maintain desired distance:

k_yaw = 150 # Rotation speed

k_throttle = 150 # Vertical speed

k_pitch = 3000 # Forward/backward (60x base gain)

deadzone_ratio = 0.3 # 30% center zone

The system prioritizes centering the marker before adjusting distance, ensuring smooth and predictable behavior.

7. Conclusion & Future Work

This project successfully demonstrates a modular, extensible architecture for vision-based autonomous drone control. The system provides robust performance across multiple control paradigms, showcasing advanced computer vision integration with real-time flight control.

Key Achievements

  • Modular Architecture: Clean separation of concerns enabling easy component swapping
  • Dual Vision Modes: ArUco markers and hand gesture recognition providing flexible control options
  • Robust Control: EMA smoothing and slew rate limiting for stable flight performance
  • Safety Features: Battery monitoring, watchdog timers, and manual override capabilities
  • Search & Recovery: Intelligent waypoint navigation when target is lost

8. Challenges and Difficulties

Throughout the development of this autonomous drone control system, we encountered several significant technical challenges that required innovative solutions and careful debugging.

warning GPS and Coordinate Limitations

Since the drone lacks built-in coordinate functionality or GPS, the drone camera must continuously track the ArUco marker to determine its current position. Our preference for the drone to follow the previous trajectory rather than planning a new one upon reacquiring the marker resulted in complicated code that required a significant amount of time to debug and organize the various control flags.

sync_problem Search Logic Inefficiencies

In the current searching logic, if the marker shifts slightly to the drone camera's left, the drone still initiates a yaw to the right, often completing nearly a full search before locating the marker. Furthermore, the drone's search is limited to rotation; consequently, if the marker is positioned too high or too low for the camera's field of view, it will not be sighted even after a full rotational search.

Current Limitations:

  • Horizontal-only search pattern (yaw rotation)
  • Fixed search direction regardless of last known position
  • Vertical blind spots outside camera field of view
  • Increased search time for markers outside rotation plane

psychology Future Improvements

While adding vertical movement commands (up and down) after a full rotation would address the vertical blind spot issue, it would substantially increase the search time. Therefore, we aim to investigate more sophisticated search methodologies that balance coverage with efficiency, potentially incorporating predictive algorithms based on last known marker position and velocity.
